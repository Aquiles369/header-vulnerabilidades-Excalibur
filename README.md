Excalibur te da un hub offline: un índice navegable que conecta tipo de vulnerabilidad, encabezado y resumen, con buscador inmediato y edición en vivo. Resultado: menos alt+tab, más impacto.</a>.**

• Búsqueda y filtros instantáneos: buscador principal por nombre de vulnerabilidad, encabezado o resumen; contadores por sección para ver volumen.<br><br>
• Edición sin fricción: Agregar ítem, Eliminar último, Alta masiva (una por línea) y Completar masivo (Nombre: resumen) para mantener vivo tu checklist.<br><br>
• Portabilidad total: Exportar JSON (con meta), Importar JSON (restaura todo) y deduplicación opcional por nombre/normalización.<br><br>
• Privacidad/resiliencia: funciona offline en tu navegador (localStorage opcional). Nada de backend, ideal para entornos aislados.<br><br>
• Listo para reportar: cada fila trae un resumen corto que te recuerda el porqué y el cómo de la relación vuln↔header.</a>.**

<br><br>

Herramienta offline en el navegador para catalogar vulnerabilidades y sus encabezados HTTP relacionados.
Crear/editar/eliminar por sección (Cliente / Servidor / Otras), alta masiva, buscar por nombre/header/resumen, contadores automáticos, y exportar/importar en JSON.

• Formato: HTML/JS estático (una sola página).

• Datos: en memoria + Export/Import JSON (y localStorage opcional para persistencia automática).

• Seguridad UI: el contenido mostrado en tablas y tooltips se escapa (no se ejecuta nada).<br><br>

- CLIENTE

• Open Redirect — Location — redirige sin validar destino.

• Email-attacks — X-Mailer — cabeceras de correo manipulables.

• Header-injections — Content-Type — inyección de headers.

• CORS — Access-Control-Allow-Origin — orígenes inseguros permitidos.

• CSRF — SameSite — sin token/mitigación suficiente.

• Cookies — Set-Cookie — flags de seguridad ausentes.

• SSTI — Content-Type — templates no saneados.

• CSP/XSS — Content-Security-Policy / X-XSS-Protection — políticas débiles/ausentes.

• WebSockets — Sec-WebSocket-Protocol — validación incorrecta.

• XXE — Content-Type: application/xml — entidades externas habilitadas.

(y más: Quick-tricks, Cache, etc.)<br><br>


- SERVIDOR

• Command Injection — CSP (contexto) — entradas no filtradas ejecutan comandos.

• CRLF / Smuggling — Transfer-Encoding — request smuggling / header injection.

• General-info — Server / X-Powered-By — fingerprinting sensible.

• IDOR / ATO — Authorization — control de acceso/autenticación débil.

• SSRF — Content-Type — server-side requests mal validadas.

• VHosts — Host — virtual hosts mal configurados.

• Crawl-fuzz — Cache-Control — endpoints internos expuestos.

• SQLi / ORM / LDAP / SAML — Content-Type / Authorization — inyecciones varias.

• Race conditions — Authorization — validaciones de estado frágiles.

• Serialización — Content-Type — deserialización insegura.<br><br>


- OTRAS VULNERABILIDADES

• Broken Links — 404 Not Found — referencias muertas.

• Online hashes cracked — Security — credenciales expuestas/comprometidas.

• API Key Leaks — Authorization / X-API-Key — claves expuestas.

• Session fixation — Set-Cookie — fijación de sesión.

• JWT — Authorization — firmas no verificadas/algoritmos débiles.

• Reverse-proxy bypass — *X-Forwarded- ** — evasión por cabeceras.

• S3/AWS — ACAO — CORS permisivo en buckets.

• Kubernetes / Docker — Authorization — malas configuraciones.

<br><br>

- Categorías 

CRUD por categoría

• Agregar ítem (pide: Vulnerabilidad, Encabezado relacionado, Resumen).

• Eliminar último (rollback rápido).

• Alta masiva (una vulnerabilidad por línea; opcional “Nombre: encabezado – resumen”).

• Completar masivo (solo descripciones/resúmenes por orden o Nombre: resumen).

• Deduplicación (case-insensitive; normaliza guiones/espacios).<br><br>



- Buscador global

• Panel superior/lat. con input que filtra por nombre, header y resumen.

• Enter salta al primer resultado; scroll suave y resaltado del ítem.

• Exportar / Importar

• Exportar JSON: incluye meta (fecha/autor/versión), secciones y filas.

• Importar JSON: restaura el estado completo.<br><br>


- Persistencia local (opcional)

• Parche para guardar/recuperar desde localStorage (incluye término de búsqueda).

• Sin parche, todo vive en memoria y se mueve vía JSON.<br><br>
