Encabezados de Seguridad


CLIENTE

Tipo de Vulnerabilidad Encabezado de Seguridad Relacionado Resumen
Open Redirect Location (header) Redirige al usuario sin verificar la validez de los destinos.
Email-attacks X-Mailer La aplicación permite manipular cabeceras de correo.
Quick-tricks Ninguno Vulnerabilidades menores que pueden ser explotadas rápidamente.
Insecure Source Code Management X-Content-Type-Options El código fuente está expuesto o mal gestionado.
Header-injections Content-Type Permite la inyección de encabezados no controlados.
CORS Access-Control-Allow-Origin Permite solicitudes de orígenes no confiables.
CSRF SameSite (cookie attribute) No implementa token CSRF o tiene protección insuficiente.
Cookies Set-Cookie Cookies sin flags de seguridad.
Cache Cache-Control, Pragma, Expires Permite manipulación de la caché de respuestas sensibles.
SSTI Content-Type Permite la inyección de plantillas sin sanitizar.
CSP Content-Security-Policy No tiene políticas de CSP adecuadas.
XSS Content-Security-Policy, X-XSS-Protection Permite la ejecución de código malicioso inyectado en entradas de usuario.
WEBSOCKETS Sec-WebSocket-Protocol No valida correctamente las solicitudes de WebSocket.
XXE Content-Type: application/xml No desactiva adecuadamente las entidades externas en XML.
Deserialization Content-Type Deserializa datos de forma insegura.

SERVIDOR

Tipo de Vulnerabilidad Encabezado de Seguridad Relacionado Resumen
Command-injection Content-Security-Policy Acepta entradas de usuario no filtradas y ejecuta comandos en el sistema.
CRLF Transfer-Encoding: chunked Permite inyecciones de encabezados a través de parámetros no sanitizados.
General-info Servidor (X-Powered-By) Exposición de información sensible sobre la configuración del servidor.
IDOR Authorization (header) Permite el acceso no autorizado a recursos sin la verificación adecuada de la autorización.
Account Takeover Authorization (header) Explotación de vulnerabilidades de autenticación para tomar control de una cuenta.
inyección DOOM Content-Type Explotación de vulnerabilidades de inyección para dañar la lógica del servidor.
Vhosts (Virtual Hosts) Host Falta de correcta configuración de virtual hosts en el servidor web.
Crawl-fuzz Cache-Control Permite el acceso no autorizado a endpoints internos del servidor.
CORS Access-Control-Allow-Origin Mal configuración de las cabeceras CORS, lo que permite acceso desde orígenes no seguros.
Business Logic Errors Authorization (header) Errores en la lógica de negocio que pueden ser explotados para obtener acceso no autorizado.
SSRF Content-Type Permite que un atacante realice solicitudes maliciosas a servicios internos.
Dns-rebinding Host Permite al atacante manipular solicitudes DNS para redirigirlas a un servidor controlado por él.
Inyección XSLT Content-Type Permite inyección de código malicioso a través de archivos XSLT.
Inyección ORM Authorization (header) Explotación de vulnerabilidades en el manejo de bases de datos mediante ORM.
Inyección LDAP Authorization (header) Permite la manipulación de consultas LDAP a través de entradas no controladas.
Inyección SAML Authorization (header) Manipulación de datos SAML para falsificar la autenticación.
SQLi Content-Type Permite la inyección de consultas SQL maliciosas.
SQL Content-Type Errores en la ejecución de consultas SQL pueden exponer la base de datos.
No/SQL Authorization (header) Vulnerabilidad en bases de datos NoSQL que permite inyecciones de código malicioso.
Parameter-pollution Content-Type Inyección de parámetros duplicados o maliciosos para alterar la lógica del servidor.
Prototype-pollution Content-Type Manipulación de prototipos en objetos JavaScript para modificar el comportamiento del servidor.
Request Smuggling Transfer-Encoding Manipulación de solicitudes HTTP para eludir las defensas del servidor.
Condición de carrera Authorization (header) Condiciones de carrera que permiten acceder a recursos sin la validación correcta.
Serialización Content-Type Deserialización insegura que permite la ejecución de código malicioso.
